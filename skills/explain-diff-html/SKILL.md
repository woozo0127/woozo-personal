# Explain Diff (HTML)

지정된 코드 변경에 대한 풍부한 인터랙티브 설명 페이지를 만든다. 설명은 한글로 작성한다 (코드 식별자와 기술 용어는 원형 유지).

## 섹션 구성

다음 섹션을 담는다:

- **배경(Background):** 이 변경과 관련된 기존 시스템을 설명한다. (이를 위해 주변 코드를 폭넓게 탐색할 것.) 독자가 얼마나 알고 있는지 모르므로, 초심자를 위한 깊은 배경(이미 익숙한 독자는 건너뛰어도 된다고 표시)을 먼저 두고, 이어서 변경에 직접 관련된 좁은 배경을 둔다.
- **직관(Intuition):** 이 변경의 핵심 직관을 설명한다. 전체 세부사항이 아니라 본질을 전달하는 데 집중한다. 토이 데이터를 이용한 구체적인 예시를 들고, 그림과 다이어그램을 아낌없이 쓴다.
- **코드(Code):** 코드 변경을 상위 수준에서 워크스루한다. 이해하기 쉬운 방식으로 변경을 묶고 순서를 정한다.
- **퀴즈(Quiz):** 이 변경의 이해도를 검증하는 문제 5개를 만든다. 난이도는 중간 — PR의 실질을 이해해야 풀 수 있되 함정 문제는 아니어야 한다. 목표는 독자가 실제로 이해했는지 스스로 확인하게 돕는 것. 인터랙티브 객관식으로 제시되며, 클릭하면 정오답 여부와 피드백이 표시된다.

## 포맷

- **HTML을 손으로 작성하지 말고, 이 SKILL.md와 같은 디렉토리에 있는 `render.py`를 사용한다.**
  CSS·JavaScript·페이지 스캐폴딩은 매 호출마다 동일하므로 렌더러가 처리한다 — 내용 spec만
  작성하면 된다. 작은 JSON spec(title, subtitle, slug, raw HTML 본문의 sections,
  정답/오답 옵션이 있는 quiz)을 임시 파일로 작성하고 실행한다:
  ```
  python3 <스킬 디렉토리>/render.py <spec.json>
  ```
  CSS, JavaScript, 페이지 스캐폴딩, 목차, 퀴즈 옵션 셔플, 날짜 접두사가 붙은 출력
  파일명을 모두 자동 처리한다. 정확한 JSON 스키마는 `render.py`의 docstring 참조.
  렌더가 끝나면 출력 경로가 stdout으로 출력된다 — 사용자에게 경로를 알려준다.
- 퀴즈 옵션 순서는 렌더러가 렌더 시점에 셔플한다 — spec에는 자연스럽게 읽히는 순서로
  쓰면 되고, 무작위로 보이게 하려고 정답 위치를 수동으로 섞을 필요 없다.
- spec의 `html` 필드는 직접 작성하는 raw HTML이다 — 코드 블록은 `<pre>`(렌더러가
  `white-space: pre-wrap` 스타일을 입힌다), 플로우 다이어그램은
  `.diagram`/`.flow`/`.box`/`.box.fail` div, 핵심 정의·엣지 케이스는 `.callout`,
  비교 표는 일반 `<table>`을 쓴다. 사용 가능한 클래스 전체는 렌더러의 CSS 참조.
- Martin Kleppmann 같은 명료함과 흐름으로, 몰입감 있는 클래식 스타일로 쓴다.
  섹션 간 전환은 매끄럽게.
- 다이어그램 팁: 소수의 다이어그램 패밀리를 정해두고 설명 전체에서 여러 케이스에
  재사용하는 것이 이상적이다. 유용한 종류:
  - 앱에서 사용자가 보는 UI를 아주 단순화한 버전 — UI 변경 설명용
  - 컴포넌트 간 데이터 흐름이나 통신을 보여주는 시스템 다이어그램 — 예시 데이터 포함
- ASCII 다이어그램은 쓰지 않는다 — 렌더러의 HTML 다이어그램 클래스를 사용한다.
